FORMS

Forms are created by using the (forms) element. When the element is typed, (name, id)attributes wld be required in the forms element and those would be functional only within the javascript. The (id) attribute is important when using labels as the (label) element with a (for) attributewould require the input data in the (id) of the forms element. Within the forms element, an element input would be required inorder for the user to input the required information into the form. After the information has been collected, a submit button would be needed to submit the form. Submit can be written in two ways, (1) either by using a (button) element and an attribute (submit) or by using an (input) element with an attribute (type="submit").

(Input) element is used within the forms element to enable the user add inputs into the forms element. With the (input) element, data types such as

  • password
  • e-mail
  • text
  • date&time
  • etc

can be collected from the user.


(Label) is an element that is used to give a title to a required information space. It goes along with the (for) attribute. The data in the (for) attribute must match the data in the (id) attribute of the (input) element. The paragraph (p) element can also be used in-place of a (label) element.

(Placeholder) is an element that is used to display a required information to the user inside on a rectangular shape where the user is meant to type in the information. With a (Placeholder), the guide would disappear as soon as the user starts typing inside the info collector.

(Value) is an attribute that does the same function as the Placeholder but the information typed inside of the shape to give the user a guide to what is needed does not disappear, the user would have to manually clear it and start typing his/her own information. It can also be used to change the a title of a button element. (Value) attribute is also used to send a required data to the server

(Textarea) element is a big textbox that is used by the user to give more information about a a topic. It is followed by (name, id, cols, and rows) the cols and rows is used to determine how many rows and column of text is needed for the information to be provided.

(Radio and checkbox) attributes are used to allow a user pick an option from a listing. The option selected would be sent to the server using the (value) attribute. The major difference between the (radio) and (checkbox) attribute is that the radio allows only one selection as long as the (name) attribute has the same data type while the checkbox allows multiple selection. (checked) attirbute is added to the input element if a particular option is to be checked by default.

(Select) element is used to great a dropdown menu where list of options awould be displayed to the user for selection. The selected option would be sent to the server when the submit button is clicked. The (option) attribute is written within the (select) element to create a list of options for the user to pick from in the dropdown.

NOTES

The syntax of css, should have a the following: selector (element that is to be styled e.g h1 element), curly braces({}), declaration block, property of the element e.g color, border,font etc, also value for the property eg the color of the color property.

There are 3 ways of using CSS, and they are

  • CSS
  • Internal CSS and
  • External CSS


INLINE CSS

Inline css is written inside the tag of the element that is required using the element (style), inside the element style, properties/attributes that are required such as font, collor, border etc would be declared. Inline Css has a downside because it wld be slow and tedious to style every single element one after the other. Although it is the strongest of all types because if all 3 types are written for a particular element, the inline would override the remaining 2. i.e the style declared in the inline will be the only active style. eg of inline css is applied below.



Inline styling was used to style this text.



INTERNAL CSS

Internal css is written inside the head element by creating the element (style), inside the style element, the element to be styled would be declared and then properties that are required for the styling such as font, color, border etc would be declared followed by the required values. Internal css also has a downside because it wld clog the html document (it is not flexible for multiple pages) as compared to the External CSS and also every single element in the document would be stlyed one after the other. For instance stlyling the h1 element, then image element and any other element that requires styling. If an element is styled, the styling affect all the elements of the same type written in any part of the HTML doc. Although it is stronger than the external css because if they are both written for a particular element, the internal css would override the external css. i.e the style declared using the internal css will be the active style. example of internal css is applied below.



THIS HEAD ELEMENT IS STYLED WITH INTERNAL CSS


EXTERNAL CSS

External css is written in a different file with the extension (.css), in the a different file document and then linked to the html page using a (link) element within the head element, (style) element is not needed, the elements to be styled are declared and then the required properties and value are also declared. For multiple pages, a link to the css should be added to the html document if the same styling is needed, instead ot copying and pasting styles as in internal css. A semi-colon is used after the property before the value is declared i.e (color: red) The below id and class heading is styled with external css.

External css can also be used to style a group of elements, a comma (,) should be added inbetween the elements e.g (h1,h2), whatever style is written inside of the group would be active for all the elements that has been grouped.

Types of Selectors

  3. ID (#name) Selector
  4. Class (.name) Selector


CSS INHERITANCE

inheritance occurs when a style is given to an element and all other element inside of the parent element takes/inherits the style of the parent element

Declaration Block

Declaration block in css starts with the braces {}. This means that all the codes or elements that would be typed inside of the braces are called declarations in the braces which signifies the block.

Declaration properties

Declaration properties are the elements that would be used for the styling, such as colors, border, background etc. all these properties will be followed by a colon (:) before the value of the property would be declared. After the declaering the value, a semi-colon should be used to close the property inorder for the style to effect on the webpage.

USING ID & CLASS

IDs are specific names given to an element. These unique names would be used to style the element in the css file. ID is mostly used to style headers, footers, menu etc. Remember the name attribute of all IDs to be used should be unique else IDs of the same name would have the same styling. In the CSS file, (#) would be written before the ID name to be used. That is the syntax for calling the id. e.g #name{ the style properties to be used}.

Class works more like IDs but Class is used when focusing on a group of elements that would have the same type of styling. It is more flexible and time-saving because all the elements that have the same class would be have the same styling, in contrast to IDs where each element would have a unique name and styled separately. In the CSS file, (.) would be written before the ID name to be used e.g .name{ the style properties to be used}. ID is mostly used to style group of headers, paragraphs, links etc

For instance in a case where there is a lot of elements of same headings e.g (10 h3<headings> in a page) class element can be used to style 3 or 4 of them within one css declaration block but using id, each of the h3 element would have a unique name and then styled separately in different blocks.

DIV AND SPAN ELEMENTS

DIV and SPAN are elements used for grouping.

Div is used to group elements/characters to be styled and the styling will take place in a declaration block and the elements within the div takes the declared style. Div is a block level element and it starts a new line when used. Span on the other hand is an inline level element that is used to style characters within the body of an element. i.e inline. The Class attribute can also be used within divs and spans to give a stlye to a particular class of the div or span. This paragraph is styled with div and also the words in italic, including this one, they are styled with the span elements.

Last Rule, Specificity, Universal Selector (*)

The last rule is where css uses the last rule and add it to the stylesheet. The last declaration of same element in css is the only that would be applied on the webpage.

Specificity is the measurement Specificity of elements in css.

Universal selector selects all the elements in the webpage and it has the least amount of Specificity. In the stylesheet it starts with (*) Selector followed by the curly braces {} then the property(ies) that needs reseting. It is mostly used to reset the default browser styles.

COLOR NAMES (PROPERTIES, RGB, RGBA,HSL, HEX(#), VS_Colors)

The color property is responsible for the color of the page or elements also background-color is also used to color the background of either the page, elements or characters.

There are 140 color names in vs code. A color-name can be used to give colors such as red, white, yellow, turquoise etc.

RGB Color is used to get extensive colors within the Red Green Blue colors by assigning values between 0-255 for each of R,G or B in a bracket. The syntax is as follows RGB(0-255,0-255,0-255), the values will be changed according to the required color. e.g RGB (135,206,235) for skyblue color.

Another color name is RGBA which is the same as RGB except for the a that is used to determine the opacity/transparency of the color, the opacity has a value of 0-1. The RGBA is mostly used within background-color property. e.g RGBA (255,248,220,0.5) cornsilk color with opacity of 0.5.

Another color value is HSL this value is uses the aspect of RGB value but the difference is instead of assigning values, HSL uses (%) to determine the color to be displayed.

Hexadecimal values is another way to insert colors using hexadecimals of alphabets between A-F (A being the least with a value 10 and F being the highest with a value 15) and numbers between 1-9. The syntax of using hex starts with a (#) then the aplhanumerical values, e.g #ff0000 for color red. Note that the hex values should not be more than 6 characters.

VS code offers color variants, it can be used typing the property color and then typing any color value, then hover on the color and then a color pallete would open up for more color and opacity options.

Pixel(px) is an absolute value of measurement(px for short) that determines the measurement of the property selected for the styling. 1 pixel is equivalent to 1 point on a screen. A pixel value stays the same across all screen types in a webpage.

A percentage value (%) is a value used in styling, the % value is a relative value of measurement that depends on the parent tags to function. It takes the percentage of the absolute measurement style.

Em is also a relative measurement value. 1em = 16px The em value depends on the parent tag values.

Rem is also a relative measurement value. The em value depends on the root tag values in the webpage i.e HTML tag.

vh & vw are absolute measurement value that takes a percentage of the browser screen. They are mostly used to with width and heigh properties to style headers and banners.

CSS BOX MODEL

Css displays elements as a box, three properties are going to be applied within the box and those properties are
1. Padding: This deals with the distance between the content and the border edge. Padding values can be written for each sides of the pad, although a shorter way of writting the values is by grouping top and bottom, giving them same value, if you'd like them to have same value, then grouping the left and right sides, also assign a single value. e.g Padding: 25px 50px (The 25px applies to the top and bottom while the 50px applies to the left and right padding.) Another way of writing the padding is by assigning different values in the padding element. e.g Padding: 10px 20px 30px 40px (10px applies to the top, 20px applies to the right, 30px applies to the bottom and then 40px applies to the left.)

2. Margin is the distance between content and the edges of the screen or some other element. Margin values can be written for each sides of the margin, although a shorter way of writting the values is by grouping top and bottom, giving them same value, if you'd like them to have same value, then grouping the left and right sides, also assign a single value. e.g Margin: 25px 50px (The 25px applies to the top and bottom while the 50px applies to the left and right padding.) Another way of writing the Margin is by assigning different values in the Margin element. e.g Margin: 10px 20px 30px 40px (10px applies to the top, 20px applies to the right, 30px applies to the bottom and then 40px applies to the left.)

Margin with a negative value overlays elements. eg. margin-top: -100px. 3. Border is the box that goes around an element. Border can be designed with different values, such as the color, width, style,radius etc. each and every design can be written on separate lines, eg.
1. border-color: green,
2. border-style: solid,
3. border-width 20px etc.
A shorter way of writing the borders is a single line is to group, the style, width and color together. The order of prefference would be width value first followed by the style and then the color. eg. border: 20px solid green.

Border-radius is used to make the edges of a border curve. The extent of the curve is dependent on the value given to the border-radius property.
Box-sizing with the property of border box

OUTLINE and OFFSET

Outline is more like a border around a content but the difference between border and outline is that with an outline-offset property, the border around the element can be moved in and out of the content, given it a different look and feel. a negaive value of outline-offset takes the border further in while a positive outline takes the border further out.


DISPLAY PROPERTY

Block and inline property. A (div) element is a block element. Block element always starts a new line and takes full width.
A (span) is an inline element. An inline doesn't start a new line and only take up as much as the content space.

When a display property is used with an assigned value, it renders the element according to the assigned value. For instance is a display peoperty with a value of inline in added to a div element, the content will be rendered with an inline style as opposed to the default property of a div. The same would happen if a display property with the value block is added to a span element. The contents would be rendered with a block style.

Horizontal centering is for aligning elements to the center. For inline elements, the text align property can be used to align the text but for block level elements, margin with a value of auto for the left and the right would align the contents of the block elements to the left.

Browsers respect top bottom margin and padding for block elements but doesn't for inline element.

list-style-type is an element used to style the numbering of un-ordered list (ul), for links in a list (ul or ol) to have top and bottom margin, first the display property has to be set to block because it is by default an inline style and browers do not render top and bottom margin for inline elements.

Inline-block is a property used with display element to make an inline element render a top and bottom margin by making the element a block property while maintaining it's inline position.

Display property set to a value of none makes an element disappear from the browser page, the element with this property would not be displayed on the webpage and devtools.

Opacity is an element that has a value between 0-1, it used to set the transparency level of an element. Just like ain rgba value of the color element, but this time, it is applied to elements.

Visibility element is used to make an element visible or hidden from the webpage but in devtools it will be evident that there is an element not displaying on the webpage.

PSEUDO CLASSES (:)

The syntax for a pseudo element would be (element:pseudo style{styling properties} e.g p:hover{styling properties}). Below a few pseudo classes would be explained with examples embedded in them.

First we would look at the Hover class (:hover) The hover is a class in the pseudo element styling that makes the state of an element change when the mouse is hovered over the element in the webpage and the chnage would be based on the pseudo property used under the the hover class. Hover can be used on any element including links (a) An example where it used on links would be seen when you hover over the header above that contains links..

Link, visited link and hover link and active link
Link pseudo class is used to design links in the webpage, given them a particular design to identify them as links. The syntax for wrriting it in the css file is (element(class/id):link{styling properties}).

Visited link pseudo class is used to stlye a link, identifying that the link has been visited before. The link could have an identifying property such as color or text design or whatever the programmer wishes to use as an identifier. The syntax would be (element(class/id):visited{styling property}).

Pseudo Hover class is the same as explained above. We would only use it here to style link elements to add a style when it is being hovered in the webpage.

Active is used to style active links in the sense that it would give a style when overed on, to show that the mouse is on the link to be clicked and the syntax would be (element(class/id):active{styling property}).
This classes has been used in the header above for the links. Explore the links on the header to see the styles.

TRANSFORM & ANIMATION

Transform property is usually used to arrange divs around the webpage, inorder to move the div to other places in the webpage, the property transform would be used with the value translate().

Transform translateX/Y is used for the direction. X with a positive value moves the div to the right while a negative value moves the div to the left. Y with a positive value moves the div to the bottom while a negative value moves the div to the top. The box below was moved with a transform property in the CSS file. The property for the syntax would be (transform: translateX/Y(Value)). The shorthand for X/Y would be transform: translate(X_value, Y_value).

Transform scale is another value used to resize the scale of the div element, i.e it makes the div smaller or bigger depending on the value used, the default value is (1), to scale smaller, the value has to be below 1 e.g (0.9 and lower), to scale bigger, the value would be above 1 e.g (1.1 and above). The syntax is (transform: scaleX/Y(value)).The shorthand for X/Y would be transform: scale(X_value, Y_value).

Transform rotateZ is used to rotate the div element based on the inputted degree. The syntax is (transform: rotateZ(Value_deg)).The shorthand would be transform: rotate(Value_deg).

Trasform SkewX/Y is used to make the div element skew/slant. The syntax is (transform: skewX/Y(Value_deg)). The syntax is (transform: skewX/Y(Value_deg)).The shorthand for X/Y would be transform: scale(X_value_deg, Y_value_deg). The below div elements has all the transform property.

TRANSITION

Transition is used to create a state of change to an element within a period of time. Change over time. Transition is used with the hover pseudo class to display the transition when the element is hovered on. Transition moves from 0-100% i.e back and forth.

Transition-property is the property used to input the required transition property such as background color, border-radius, color, font-size, transform etc. The properties required when hovered would be written inside the pseudo hover class of the element. An example is given below.

div.TransI {

width: 150px;
height: 200px;
font-weight: bold;
display: inline-block;
}
div#wife {

transition-timing-function: linear;
}
div#wife:hover {

background-size: cover;
border-radius: 50%;
}

Flex direction makes the content of the flex align either in a column or a row. Although column is the default one. We also have values such as row-reverse while reverses the order of the items in a row of the flex container. Also column-reverse whihc also reverses the items but in a column. The syntax is given below.
flex-direction: row;
flex-direction: row-reverse;
flex-direction: column;
flex-direction: column-reverse;
You use the flex direction and wrap only after the display has been set to flex.

Flex Wrap is a property that pushes the flex items to the next line so as not to have continous unbroken line of characters. The syntax is:
flex-wrap: wrap; (This wraps the content to the next line)
flex-wrap: nowrap; (This is the default value of the html document. The items stay on a single line without moving to the next.)
flex-wrap: wrap-reverse; (This value wraps the items in a reverse order. It takes the test above the line of characters instead of below it. Inorder words the line before and not after.)

There are two main elements that are used to justify and align flex items inside the container. These elements contain 3 main values and 2 special values which aligns them in a very different pattern. The first would be:
justify-content: this property aligns the flex items horizontally and the 3 main values are: Flex-end (right hand side),
Flex-Start(Left hand side) and
Center(middle).
The special values are:
Space-between (creates equal spaces between the flex items.) and
Space-around (creates spaces around all the items in the conatainer, including the edges, more like margin all around.)
align-items: this property aligns items vertically and the 3 main values are the same as does with justify-content. That is:
Flex-end (bottom),
Flex-Start(top) and
Center(middle).
The special values are:
Baseline (This value keeps the items on the same baseline no matter the font size or height of the property.) and
Stretch (This is the default value of the flex where the item is streched from top to botton of the flex container)

Flex Item Order is used to change how the items are positioned or ordered in the container. The syntax would be to add an inline style to the items and use the property order: ; the value would range from 1 to the number of items that are in the container. 0 can not be used as it is the default value that keeps the items in their original order. SHRINK/GROW AND BASIS

These properties are responsible for the responsiveness of flexbox.

Flex Basis defines the minimum width of a flex item. The value can be any number in pixels.

Flex-grow defines the same amount of extra space between items in a flex-box. Set Flex grow to 1 for all items in the flex if they are to have the same amount of extra space but if one item will have a defined amount extra space, set the value of the item to 1 while others retain the default value which is 0.

Flex-shrink defines the rate of shrinkage of each item relative to the rest of the items and the default value is 1. Set the value of flex shrink to zero if a particular item in the box will not be shrinked when the minimum shrink of the webpage is reached. A value above 1 makes the item shrink as many times as the defined value.For instance, if the value of a flex-shrink is set to 3, it means the item that has that value would shrink 3 times faster than the remaining items in the flex.
These flex itmes are very important as it helps the webpage to be responsive to any device or size of screen that it would be displayed on. Below is an example of the effect of the flex grow,shrink and basis properties.

GRID

Grid is a display type that can be used to activate certain layout features on a container element. The major difference between grids and flexbox: grid is more manual that flexbox, grid focuses on both width and height while flex focuses majorly on the width of the flex items.

Template rows and columns can be created in the container, the properties for creating rows and columns is grid-template-columns and grid-template-rows. grid-template-columns: 50px 50px 10px; This property specifies the number of rows/columns that the grid container would be divided into. The number of values inputted determines the number of rows/columns that will be in the grid. Using auto as the value for rows and columns would be the perfect value, as it would make the grid responsive to any device and also divides the items evenly in the grid container. Note the number of auto defined in the value, determines the number of rows/columns that would be created.
justify-content: this property aligns the grid items horizontally and the 3 main values are: end (right hand side),
Start(Left hand side) and
Center(middle).
The special values are:
Space-evenly (creates even spaces between the grid items.) and
Space-around (creates spaces around all the items in the conatainer, including the edges, more like margin all around.)
align-content: this property aligns items vertically and the 3 main values are the same as does with justify-content. That is:
end (bottom),
Start(top) and
Center(middle).

grid-column-start/end is used to determine a grid item's size and location within the grid by contributing a line, a span, or nothing (automatic) to its grid placement.

grid-area is a shorthand for grid-column/row-start/end that determine a grid item's size (by adding span after the slash before the next value) and location within the grid by contributing a line, a span, or nothing (automatic) to its grid placement. Shorthand for 'grid-row-start', 'grid-column-start', 'grid-row-end', and 'grid-column-end'.
